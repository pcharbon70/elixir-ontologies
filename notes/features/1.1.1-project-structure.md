# Feature 1.1.1: Extend Existing Project Structure

## Problem Statement

The elixir-ontologies project currently contains only ontology definitions (Turtle files) and documentation. To implement the Knowledge Graph tooling planned in Phase 1-10, we need to extend the project with RDF dependencies and create the module structure for code analysis, graph operations, and RDF construction.

## Solution Overview

Extend the existing mix.exs to add RDF.ex and SPARQL dependencies, then create the directory structure for the analyzer modules:

- `lib/elixir_ontologies/graph/` - Graph CRUD operations
- `lib/elixir_ontologies/analyzer/` - Code analysis (AST parsing, extraction)
- `lib/elixir_ontologies/builders/` - RDF triple construction
- `lib/elixir_ontologies/extractors/` - AST-to-RDF extraction

## Technical Details

### Dependencies Added

```elixir
{:rdf, "~> 2.0"}           # Core RDF library
{:sparql, "~> 0.3", optional: true}  # SPARQL query support
```

### Directory Structure

```
lib/elixir_ontologies/
├── graph/           # Graph wrapper and CRUD operations
├── analyzer/        # AST parsing and code analysis
├── builders/        # RDF triple construction helpers
└── extractors/      # AST-to-RDF extraction logic
```
