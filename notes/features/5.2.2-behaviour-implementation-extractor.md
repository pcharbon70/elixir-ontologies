# Feature: 5.2.2 Behaviour Implementation Extractor

## Problem Statement

Need to extract behaviour implementations from Elixir modules - the `@behaviour ModuleName` declarations and matching callback implementations. This complements the Behaviour Definition Extractor (5.2.1) which extracts the behaviour definitions.

## Solution Overview

Extend the `Behaviour` extractor module with implementation extraction:
1. Detect `@behaviour ModuleName` declarations in module body
2. Create `BehaviourImplementation` relationship structure
3. Match implemented functions to behaviour callbacks via `implementsCallback`
4. Detect `defoverridable` for default implementations
5. Link overrides via `overridesDefault`

## AST Structure Analysis

```elixir
# @behaviour creates this AST:
{:@, meta, [{:behaviour, _, [{:__aliases__, _, [:ModuleName]}]}]}

# defoverridable creates:
{:defoverridable, meta, [[name: arity, ...]]}

# or with module reference:
{:defoverridable, meta, [{:__aliases__, _, [:BehaviourModule]}]}
```

### New Struct Fields

**BehaviourImplementation:**
- `:behaviour` - The behaviour module being implemented (alias atom or list)
- `:location` - Source location of @behaviour declaration
- `:implemented_callbacks` - List of {name, arity} that match callbacks
- `:overridable_functions` - List of {name, arity} marked defoverridable

**Module Implementation Result:**
- `:behaviours` - List of @behaviour declarations
- `:overridables` - List of defoverridable entries
- `:functions` - List of defined functions for matching
