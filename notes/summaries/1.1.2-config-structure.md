## Overview

Implemented the `ElixirOntologies.Config` module providing centralized configuration for the Knowledge Graph tooling.

## Changes Made

### 1. Config Module (`lib/elixir_ontologies/config.ex`)

Created a complete configuration module with:

**Struct Fields:**
- `base_iri` - Base IRI for generated RDF resources (default: `"https://example.org/code#"`)
- `include_source_text` - Whether to include source code (default: `false`)
- `include_git_info` - Whether to include git info (default: `true`)
- `output_format` - Output format: `:turtle`, `:ntriples`, or `:jsonld` (default: `:turtle`)

**Functions:**
- `Config.default/0` - Returns default configuration
- `Config.merge/2` - Merges user options with base config
- `Config.validate/1` - Validates config, returns `{:ok, config}` or `{:error, reasons}`
- `Config.validate!/1` - Validates config, raises on error
- `Config.new/1` - Convenience function creating validated config from options

### 2. Tests (`test/elixir_ontologies/config_test.exs`)

14 comprehensive tests covering:
- Default value creation
- Option merging (single and multiple options)
- Unknown option handling
- Validation of all fields
- Multiple error detection
- Convenience function behavior
