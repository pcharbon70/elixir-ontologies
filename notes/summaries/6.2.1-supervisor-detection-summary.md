# Summary: 6.2.1 Supervisor Detection

## Implementation

Created `lib/elixir_ontologies/extractors/otp/supervisor.ex` with:

### Supervisor Struct
```elixir
%Supervisor{
  supervisor_type: :supervisor | :dynamic_supervisor,
  detection_method: :use | :behaviour,
  use_options: keyword() | nil,
  location: SourceLocation.t() | nil,
  metadata: map()
}
```

### Detection Functions
- `supervisor?/1` - Returns true if module implements any Supervisor behaviour
- `use_supervisor?/1` - Detects `use Supervisor` AST pattern
- `use_dynamic_supervisor?/1` - Detects `use DynamicSupervisor` AST pattern
- `behaviour_supervisor?/1` - Detects `@behaviour Supervisor` AST pattern
- `behaviour_dynamic_supervisor?/1` - Detects `@behaviour DynamicSupervisor` AST pattern

### Extraction Functions
- `extract/2` - Returns `{:ok, result}` or `{:error, reason}`
- `extract!/2` - Returns result or raises ArgumentError

### Utility Functions
- `supervisor_type/1` - Returns `:supervisor` or `:dynamic_supervisor` or nil
- `detection_method/1` - Returns `:use` or `:behaviour` or nil
- `use_options/1` - Extracts options from use statement
- `dynamic_supervisor?/1` - Returns true for DynamicSupervisor modules
- `otp_behaviour/0` - Returns `:supervisor` (for consistency with GenServer extractor)
